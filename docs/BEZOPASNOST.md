# Безопасность аккаунта и устойчивость Prime Mafia

## Главное: бот ≠ ваш личный Telegram

Prime Mafia работает через **бот-токен** от [@BotFather](https://t.me/BotFather). Сообщения игрокам идут **от имени бота**, не от вашего личного аккаунта.

| Риск | Личный аккаунт | Бот |
|------|----------------|-----|
| Массовая рассылка приглашений | Высокий риск блокировки | Нормальная практика, если есть отписка `/stop` |
| Токен утёк | — | Перевыпустить в BotFather → обновить `TELEGRAM_TOKEN` в Railway |
| Блокировка | Потеря личного TG | Обычно блокируют бота, не владельца |

**Не используйте** userbot / личный аккаунт для рассылок — только официальный Bot API.

## Переменные окружения (Railway)

```env
TELEGRAM_TOKEN=...          # только на сервере, не в git
SUPABASE_KEY=...            # только service_role secret, не anon
ADMIN_TG_ID=123456789       # основной администратор (заявки, загрузка карт)
BACKUP_ADMIN_TG_IDS=111,222 # резервные админы через запятую
ENFORCE_TRIAL_LIMITS=true   # в проде — включить лимиты теста
```

**Не ставить на Railway:**

```env
MINIAPP_DEV_TG_ID=...        # удалить — dev-обход авторизации mini app
ALLOW_MINIAPP_DEV_BYPASS=... # только локально, если нужен обход без Telegram
```

- Dev-обход mini app: только локально при `ALLOW_MINIAPP_DEV_BYPASS=true`. На Railway — **всегда выключен**.

- `ADMIN_TG_ID` — ваш Telegram numeric id (узнать: @userinfobot).
- `BACKUP_ADMIN_TG_IDS` — 1–2 доверенных человека: доступ к админ-функциям, если основной аккаунт недоступен.
- Токен бота **никогда** не коммитить в репозиторий.

## Резервный администратор

В коде `isAdmin()` проверяет `ADMIN_TG_ID` и список `BACKUP_ADMIN_TG_IDS`.

Рекомендация:
1. Добавить резервного админа (партнёр / техспециалист).
2. Держать вторую копию доступа к Railway и Supabase.
3. Экспорт `.env` в защищённый менеджер паролей.

## Как не получить блокировку бота

1. **Отписка** — в каждой рассылке: «/stop или стоп» (реализовано).
2. **Пауза между сообщениями** — ~55 ms (реализовано в `lib/rassylka.js`).
3. **Только база клуба** — не покупать списки, не слать незнакомым.
4. **Не спамить** — одно приглашение на анонс / игру, не дублировать каждый час.
5. **Жалобы** — если игрок нажал «стоп», бот больше не шлёт приглашения.

## Данные и Supabase

- Игры, рейтинг, клубы — в Supabase; перезапуск бота их не стирает.
- Регулярные бэкапы — в панели Supabase → Database → Backups (на платном плане) или pg_dump по расписанию.
- **Перед гостевыми клубами:** выполнить `supabase/enable_rls_club_isolation.sql` — RLS по `klub_id`, anon/authenticated не видят чужие данные.
- В Railway `SUPABASE_KEY` = **service_role** (бот обходит RLS). Anon-ключ в боте — ошибка, процесс не стартует.

## Публикация итогов в группу / канал

См. инструкцию в боте: «📖 Как пользоваться» → собственник → раздел «Публикация итогов».

Кратко:
1. Добавить бота в группу или канал (в канале — права администратора на публикацию).
2. ⚙️ Настройки клуба → 📢 Привязать группу → переслать сообщение из чата.
3. После игры — «📢 Отправить в группу клуба» или автопубликация в настройках.

## Если бота заблокировали

1. Написать [@BotSupport](https://t.me/BotSupport) через BotFather.
2. Создать **нового** бота, перенести `TELEGRAM_TOKEN`, обновить `@username` в материалах клубов.
3. Данные клубов в Supabase сохранятся — меняется только точка входа в Telegram.
