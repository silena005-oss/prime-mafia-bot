# Бонусы, подарки и отчёты рейтинга

## Логика (как договорились)

### 1. Лучший игрок вечера → выбор карты

```
Игровой вечер завершён
    → ведущий/система выбирает «лучший игрок вечера»
    → игроку начисляется бонус типа vybor_karty (выбор роли на следующую игру)
    → бонус появляется в mini app: вкладка «Мои бонусы»
    → статус: active (не использован)
```

На следующем вечере:

```
Ведущий: игровой вечер / панель игры
    → нажимает на ник игрока за столом
    → «Использовать бонус»
    → список доступных бонусов игрока (иммунитет, выбор карты, …)
    → подтверждение → бонус spent, эффект применяется в текущей игре
```

### 2. Источники бонусов

| Источник | Примеры |
|----------|---------|
| **Клуб** | лучший игрок вечера, турнир, акция клуба |
| **Игрок** | подарок другому игроку (косметика / бонус) |
| **Система** | достижение, промокод, компенсация поддержки |

### 3. Типы бонусов (v1)

| `tip` | Эффект в игре |
|-------|----------------|
| `vybor_karty` | Игрок выбирает роль из разрешённого пула клуба до раздачи |
| `immunitet_golos` | Нельзя выставить на голосование (1 раз / до конца дня) |
| `immunitet_noch` | Защита от ночного выстрела (по правилам клуба) |
| `podarok_kosmetika` | Рамка профиля / бейдж в mini app (без влияния на игру) |

Клуб задаёт **какие типы разрешены** в `kluby.nastroyki.bonusy`.

### 4. Отчёт рейтинга (mini app)

**Игрок:** карточка-сводка — очки, игры, победы, место в клубе, лучшая игра, последние 5 игр, активные бонусы.

**Клуб:** топ-10, динамика за месяц, лучший игрок вечера (если назначен), экспорт/шеринг (картинка для TG).

Отдельный экран `/miniapp#report` или вкладка «Отчёт» в inspector.

### 5. Стилизация

- Тема клуба (Ellada, Sochi, …) → CSS variables + карты ролей в боте.
- Отчёт и «Мои бонусы» используют клубные цвета и логотип.
- Карты Ellada: загрузка через `/club_cards` + пресет в `clubs/prime-mafia-ellada/`.

## Таблица `igrovye_bonusy`

См. `supabase/add_igrovye_bonusy.sql`.

Поля: `igrok_id`, `klub_id`, `tip`, `status` (active/used/expired), `istochnik`, `meta` (json: vybrannaya_rol, vecher_id, …), `istekaet`.

---

## API mini app (расширение)

`GET /api/miniapp/state` → добавить:

```json
{
  "bonuses": { "active": [...], "used_count": 3 },
  "rating_report": { "player": {...}, "club": {...} }
}
```

`POST /api/miniapp/action`:

- `pick_role_bonus` — игрок выбирает карту из бонуса
- `use_player_bonus` — ведущий (с проверкой роли)

---

## Бот (ведущий)

Callback: `igrok_menu_{kod}_{nomer}` → карточка игрока → «Использовать бонус».

Существующий иммунитет (`immunitet_golos` на игроке) связываем с spent-бонусом из `igrovye_bonusy`.

---

## Очередь разработки

1. SQL + начисление «лучший игрок вечера» после завершения вечера
2. Mini app: вкладка «Мои бонусы» + улучшенный отчёт рейтинга
3. Бот: меню игрока → использовать бонус
4. Подарки игрок→игрок и косметика
5. Ellada: карты + тема после загрузки ассетов
